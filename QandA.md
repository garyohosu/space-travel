# QandA.md

`SPEC.md`作成にあたり、`memo.md`の記述と優先順位ルールだけでは判断しきれず、ユーザー確認が望ましい論点をまとめる。

作業を止めない方針のため、`SPEC.md`側には暫定の判断を反映済み(32章「仮決定事項」)。ここに挙げた項目は、いずれ確認・修正が必要になりそうなものに絞っている。

## 1. 「Sign in with ChatGPT」の技術的実現性

`SPEC.md` 20章(SEC-002)に記載した「Sign in with ChatGPT」によるランキング登録は、`memo.md`の要望をそのまま仕様化したものであり、ChatGPT Sites側で実際にどのような認証APIが提供されるかは未確認。実装フェーズでChatGPT Sitesの認証機能仕様を確認し、必要なら`SPEC.md` 20章を更新してほしい。

## 2. GitHub Pages版に将来ランキングを持たせるか

`memo.md`は「外部バックエンドを追加しない限り世界ランキングを表示しない」としており、`SPEC.md`もそれに従った(DEP-002)。ただし、将来的に無料枠の外部バックエンド(Firebase等)を追加してGitHub Pages版にもランキングを持たせる可能性があるかどうかは`memo.md`に記述がない。方針次第で29章「対象外機能」の扱いが変わる。

## 3. 旧操作案(C・P・F5・F9・Tab)を正式機能として残すか

`memo.md`前半(現代版アレンジ仕様案)にあった以下の操作は、後発の得点・HUD仕様には言及がない。

* C:表示モード切り替え(船体固定/方角固定)
* P:フォトモード
* F5/F9:クイックセーブ/クイックロード
* Tab:近隣天体の巡回選択

`SPEC.md`では「後発仕様と衝突しないため残す」という判断で仮に正式仕様へ含めた(32章)。自動保存・自動再開(SAV-006)が導入された今、F5/F9のような手動セーブ機能が本当に必要かは、ユーザーの意図次第。不要であれば`SPEC.md`から該当箇所を削除する。

## 4. 地球の着陸基本得点

`memo.md`の得点表(難易度1〜4、巨大惑星観測)に地球が含まれておらず、基本点が不明。`SPEC.md`では月と同格(難易度1相当、100〜150点)を仮の値とした(32章、33章)。実際にどの程度の得点にすべきかはバランス調整が必要で、`memo.md`側での明記がないため確認したい。

## 5. フェーズ2・3天体の実装時期

`SPEC.md` 6章で天体をフェーズ1〜3に分けたが、これは`memo.md`「天体ビジュアル方針」章の記述をそのまま踏襲したものであり、実際の開発スケジュール(いつまでにフェーズ2を実装するか等)は`memo.md`に記載がない。開発計画を別途立てる際に確認したい。
