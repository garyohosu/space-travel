## 2. ゲーム概要

**GEN-001** 本作は、太陽系を自由に航行し、惑星・衛星への着陸または接近観測を楽しむ宇宙旅行ゲームとする。戦闘・武器・敵対NPC・資源採掘・課金要素は実装しない。

**GEN-002** 対象ユーザーは、キーボード操作可能なPCブラウザのユーザーを主とし、スマートフォン・タブレットのタッチ操作ユーザーも副次的な対象とする。年齢層は問わないが、説教くさい警告文やペナルティの重さでプレイヤーを萎縮させない方針とする。

**GEN-003** 1回のプレイ単位を「航海」と呼ぶ。航海は地球付近から開始し、宇宙船が破壊される、または他の航行不能条件に達するまで継続する。

**GEN-004** 航海の流れ:

1. タイトル画面から「新しい航海を開始」または「続きから」を選ぶ
2. 地球付近から航行を開始する
3. プレイヤーは任意の順序で惑星・衛星へ接近し、着陸または接近観測を行う
4. 到達ごとに得点が加算される
5. 宇宙船が破壊される、または航行不能になった時点で航海が終了し、結果画面を表示する
6. 結果画面から次の航海を開始するか、記録・ランキングを確認するか選べる

**GEN-005** ゲーム全体には最終クリア条件を設けない。すべての天体を訪問済みにしてもゲームは終了せず、プレイヤーは新しい航海を何度でも開始できる。

**GEN-006** 個々の航海には得点・結果が存在する(スコアアタック的性質)が、通常航行中はタイムリミットを設けず、急かされない自由な宇宙旅行として体験できるようにする。得点表示は控えめに行い、演出は落ち着いたものとする(詳細は20章・21章)。

**GEN-007** 戦闘、敵、武器、防具、対人対戦、資源採掘、クラフト、課金要素、広告は初期版で実装しない(詳細は26章)。

## 3. 対象環境

**ENV-001** 本ゲームはインストール不要のブラウザゲームとして提供する。常駐サーバー(Python/PHP/Node.jsサーバーやデータベースサーバーなど)を必要としない静的Webアプリとして実装する。

**ENV-002** 公開先は以下の2系統とする。

* **DEP-001** ChatGPT Sites:主たる公開先。ゲーム本体・天体画像・音声・ブラウザ保存・D1による世界ランキングを1つのSiteで提供する。
* **DEP-002** GitHub Pages:初期版では世界ランキングを実装しない簡易版として公開する。提供する機能はゲーム本体、自己最高得点、訪問済み天体、航海履歴、操作設定、音量設定、ブラウザ内のローカル記録とする。世界ランキングはD1を利用できるChatGPT Sites版の機能とし、初期版の必須要件には含めない。

**DEP-003** 将来拡張:外部バックエンドを追加した場合、GitHub Pages版にも世界ランキング機能を提供できる。(Firebase、Supabase、Cloudflare Workersなどが候補になり得るが、初期版の対象外とする)

**ENV-003** 対応ブラウザ:

* PC:Google Chrome最新版、Microsoft Edge最新版、Firefox最新版、Safari(macOS)最新版
* スマートフォン・タブレット:Android版Google Chrome最新版、Android版Microsoft Edge最新版、iPhone/iPad版Safari最新版

**ENV-004** 主な対象はPCブラウザとするが、スマートフォン・タブレットでもキーボードなしですべての操作を完結できることを必須とする(詳細は22章)。画面幅360ピクセル程度の端末でも主要な操作ボタン・情報がはみ出さないようにする。

**ENV-005** 画面サイズに応じたレスポンシブ表示に対応する。PC・タブレット・スマートフォンでそれぞれ専用レイアウトへ切り替え、単純な縮小表示は行わない。

**ENV-006** スマートフォンでは横向き(ランドスケープ)表示を推奨する。縦向きで開始した場合は「スマートフォンを横向きにすると、より遊びやすくなります」という短い案内を表示するが、画面の向きを強制的に固定しない。縦向きでもメニュー・航海記録・ランキング・設定は利用できる。

## 4. 技術構成

**ENV-010** 使用する基本技術:

* HTML / CSS / JavaScriptまたはTypeScript
* 描画:Canvas 2DまたはWebGL。星空・天体の発光・環・粒子表現などにThree.jsを利用してよい
* 演出:GSAP(画面遷移・得点表示・カウントダウンなどのアニメーション)
* 音響:Howler.js(効果音・BGMの読み込み、音量制御、ループ、フェード)
* アイコン:Lucide(主要操作アイコン。日本語ラベルを必ず併記する)
* 保存:LocalStorageまたはIndexedDB
* ChatGPT Sites版のみ:D1リレーショナルデータベース(世界ランキング用)

**ENV-011** ゲームの物理計算(航行・重力・衝突判定)は描画ライブラリに依存させず、ゲームロジックとして分離する。Three.jsなどは表示にのみ用いる。

**ENV-012** ビルドツールを使用する場合も、最終的にHTML・CSS・JavaScript・画像・音声などの静的ファイルとして出力できる構成にする。GitHub Pagesへはリポジトリ名を含むサブディレクトリから公開されることを想定し、絶対パスではなく相対パスまたは適切なベースパスを使用する。ビルドが必要な場合はGitHub Actionsで自動ビルド・公開する。

**ENV-013** CDNから読み込む外部ライブラリには必ず明示的なバージョン番号を指定する。`latest`の指定は禁止する。同一ライブラリの本体・追加モジュールは同じバージョン・同じCDNから読み込み、複数CDNへ分散させない。実装時点の安定版を確認し、バージョンを固定すること。

**ENV-014** CDN読み込み失敗時のフォールバック(ERR-005・ERR-006と対応):ゲーム全体を停止せず、利用できない外部ライブラリの機能をCSS、文字・簡易記号、またはブラウザ標準APIへ切り替える。詳細な障害時の共通方針はERR-005、各障害への対応方針はERR-006を参照する。

* GSAP失敗時:CSSアニメーションへ切り替える
* Lucide失敗時:文字または簡易記号を表示する
* Howler.js失敗時:ブラウザ標準のAudio APIを試みる
* Three.js失敗時:Canvas 2Dの簡易背景へ切り替える

使用するCDN・ライブラリ名・バージョン・ライセンスはREADMEとクレジット画面へ記載する。

## 5. ゲーム状態

**GEN-010** 最低限、以下のゲーム状態を定義する。各状態で受け付ける操作と、停止する処理を明確に区別する。

| 状態 | 概要 | 物理演算 | 主なキー操作 |
|---|---|---|---|
| 起動中 | 初回読み込み演出 | 停止 | なし |
| タイトル | タイトル画面 | 停止 | Enter/クリックで開始 |
| 新規航海開始 | 航海の初期化処理 | 停止 | なし(自動遷移) |
| 航行中 | 通常のゲームプレイ | 実行 | W/S/A/D/Q/E/マウス操作等 |
| 一時停止 | Spaceによる一時停止 | 停止 | Spaceで復帰、Escでメニュー |
| マップ表示 | Mキーで開く全画面パネル | 停止 | 上記+パネル操作キー |
| 航海記録表示 | Hキーで開く全画面パネル | 停止 | 同上 |
| ランキング表示 | Lキーで開く全画面パネル | 停止 | 同上 |
| 設定表示 | 設定パネル | 停止 | 同上 |
| フォトモード | Pキーで開く撮影用表示(UI-006) | 停止 | カメラ操作、Escで終了 |
| 着陸成功演出 | 着陸成功時の短い演出 | 停止(演出中のみ) | 入力は一時無効 |
| 接近観測成功演出 | 巨大惑星観測成功時の演出 | 停止(演出中のみ) | 入力は一時無効 |
| 航海終了 | 爆発演出+結果画面 | 停止 | 結果画面のボタン操作 |
| 再開カウントダウン | パネルを閉じた直後 | 停止 | 入力を受け付けない |

**GEN-011** マップ表示・航海記録表示・ランキング表示・設定表示・フォトモード・一時停止の各状態では、以下をすべて停止する。

* 宇宙船の移動
* 天体の公転
* 重力計算
* 衝突判定
* 航海時間の加算
* キーボード/タッチによる操縦入力

**GEN-012** 上記のパネルをEscキーまたは戻るボタンで閉じた直後は、直ちに航行を再開せず「3・2・1・航行再開」の短いカウントダウンを表示してから操縦入力を受け付ける。これは、パネルを閉じた直後の誤操作や重力による事故を防ぐための必須要件である。

**GEN-013** 着陸成功演出・接近観測成功演出・航海終了の爆発演出の間は、演出の必要最小限の時間だけ操縦入力を無効にする。長時間ゲームを止めてはならない(演出は数秒程度に収める)。

## 6. 太陽系と天体

**CEL-001** 太陽系の天体データは、名称・種別(恒星/惑星/準惑星/衛星)・軌道半径・質量・表示半径・公転周期・着陸可否・難易度・基本得点をひとまとめにした設定データとして外部化する(コードへ直接埋め込まない)。

**CEL-002** 惑星・衛星の軌道は円軌道に簡略化する(オリジナル版の仕様を踏襲)。軌道半径・質量は実際の天文スケールに比例させるが、ゲームとして遊びやすい縮尺・速度に調整してよい。

**CEL-003** 初期リリース(フェーズ1・必須)で実装する天体。近距離・中距離・遠距離・巨大惑星観測・衛星着陸のすべてを一通り体験できる構成とする。

* 太陽、水星、金星、地球、月、火星
* フォボス、ダイモス(火星の衛星)
* 木星、イオ、エウロパ、ガニメデ、カリスト(木星のガリレオ衛星)
* 土星、タイタン、エンケラドゥス
* 天王星、海王星、トリトン
* 冥王星

**CEL-004** フェーズ2(初期リリース後の拡張候補。日程は本文書では管理しない):

* レア、イアペトゥス、ディオネ、テティス、ミマス(土星の衛星)
* チタニア、オベロン、アリエル、ウンブリエル(天王星の衛星)
* カロン(冥王星の衛星)

**CEL-005** フェーズ3(追加天体拡張候補):

* ケレス、エリス、ハウメア、マケマケ(準惑星)
* その他の主要衛星
* 小惑星、彗星

**CEL-006** フェーズ2・フェーズ3は初期リリースの完成条件へ含めない。ただし、後から天体を追加できるよう、天体名・画像・質量・半径・軌道・難易度・得点・着陸条件などをCEL-001の設定データ形式で管理する。得点設定(14章)はフェーズを問わず先にデータ定義してよいが、実装(表示・到達判定)されていない天体にはプレイヤーが到達できないため、受け入れ条件はフェーズ1の実装天体に限定する。開発スケジュールは本仕様書ではなく別の開発計画書で管理する。

**CEL-007** 惑星を選択すると、その惑星と主要衛星をまとめて「惑星系」として天体一覧・航海記録画面へ表示する。惑星系の分類は以下とする。

* 地球系(地球、月)
* 火星系(火星、フォボス、ダイモス)
* 木星系(木星、イオ、エウロパ、ガニメデ、カリスト)
* 土星系(土星、タイタン、エンケラドゥス、レア、イアペトゥス、ディオネ、テティス、ミマス)
* 天王星系(天王星、チタニア、オベロン、アリエル、ウンブリエル)
* 海王星系(海王星、トリトン)
* 冥王星系(冥王星、カロン)
* その他の天体(準惑星、小惑星、彗星などフェーズ3拡張分)

## 7. 物理モデル

**PHY-001** 座標系は、宇宙船に最も強い重力を及ぼす天体を原点とする相対座標系を用い、数値安定性を保つ(オリジナル版の手法を踏襲)。画面表示上は宇宙船を画面中央付近に固定する。

**PHY-002** 宇宙船へ影響する重力は、逆二乗則で計算される「その時点で最も強い引力を持つ天体1つ」のみとする(複数天体の合成は行わない、簡易重力モデル)。

**PHY-003** 宇宙船の状態量:位置、速度ベクトル、姿勢角(向き)。

* W:前進方向へ一定の加速度で加速(噴射)
* S:逆噴射(減速方向へ加速)
* A/D:左右旋回(姿勢角の変更)

加速度は物理シミュレーション上は固定値とし、画面上の見かけの動きは航行スケール(NAV-001)に応じて変化する。

**PHY-004** 表示モードは以下の2種を提供し、初期設定は「船体固定モード」とする。

* 船体固定モード:宇宙船は画面中央で上向きに固定され、周囲の天体・軌道が回転する(オリジナル版の再現)
* 方角固定モード:宇宙空間の方角を固定し、宇宙船自体が画面上で回転する(現代的なゲームに近い表示)

表示モードの切り替えはCキーで行う(物理計算そのものには影響しない、表示のみの切り替え)。設定画面からも切り替え可能とし、選択状態はブラウザへ保存する(SAV-001)。スマートフォンでの切り替え方法はMOB-012を参照。

**PHY-005** 衝突判定・着陸判定は、天体表面のラインを宇宙船が通過する際の相対速度によって行う。安全速度以内であれば着陸成功、それを超えていれば衝突(航海終了)とする。安全速度・危険速度の具体的な数値は天体ごとの設定データとし、初期値は14章の「調整可能パラメータ」にまとめる。

**PHY-006** 太陽への接近は、一定距離以下で「危険接近」として航海終了条件になる(GEN-014参照)。危険距離の具体値は調整可能パラメータとする。

**PHY-007** シミュレーション時間(航海時間)は、航行中のみ加算し、パネル表示中や一時停止中は加算しない。

## 8. 航行スケールとカメラ

**NAV-001** 「航行スケール」は、宇宙船をどの距離単位で操作しているかを示すゲーム上の機能であり、単なる画面拡大とは異なる。次の段階を用意する:着陸/接近/惑星圏/惑星間/太陽系。

* 着陸スケール:細かな速度調整がしやすい
* 太陽系スケール:長距離を短時間で移動できるが、目的地を通過しやすい(オーバーシュートのリスク)

スケール変更時は画面を急に切り替えず、滑らかにズームさせる。現在の航行スケールは下部操作盤へ常時短く表示する。

**NAV-002** 航行スケールとカメラズームは明確に別機能として扱う。`memo.md`内に両者を混同した記述はないが、誤解が生じやすいため本仕様で明確化する。

* Q/E:航行スケールの縮小・拡大(ゲーム上の距離単位を変える)
* マウスホイールまたはピンチ操作:カメラの表示倍率のみを変える(ゲーム上の速度・距離には影響しない)

カメラズームは一定時間操作がなければ、現在の航行スケールに適した倍率へゆっくり戻してよい。

**NAV-003** 目的地を選択している場合、画面端に目的地方向を示す小さなマーカーを表示する(画面外の天体に対しても旋回方向が分かるようにする)。常時表示する照準器は置かない。

**NAV-004** 航行支援(任意でオン・オフ可能な補助機能):

* 軌道予測線:現在の速度と重力からのおおよその進路を表示する。噴射や重力対象の変化に応じて随時更新する
* 接近予測:目的地への最接近予測地点を表示し、通過しそうな場合は「通過予定」と表示する
* 着陸支援:安全な着陸速度・進入方向を表示し、速度超過時は「減速してください」と表示する。着陸可能範囲に入ると天体表面の安全進入範囲を淡く表示する
* 姿勢安定:キーを離した際に旋回をゆっくり止める補助。オリジナルに近い慣性操作を好む場合は無効化できる

## 9. 操作仕様

### 9.1 PC操作

**CTL-001** 基本操作:

| キー | 動作 |
|---|---|
| W | 前進噴射 |
| S | 逆噴射 |
| A | 左旋回 |
| D | 右旋回 |
| Q | 航行スケール縮小 |
| E | 航行スケール拡大 |
| マウスホイール | カメラズーム |
| マウス左クリック | 天体を選択 |
| Tab | 近くの天体を順番に選択 |
| M | 太陽系マップ(全画面パネル) |
| H | 航海記録(全画面パネル) |
| L | 世界ランキング(全画面パネル) |
| C | 表示モード切り替え(船体固定/方角固定) |
| P | フォトモード |
| F1 | 操作説明 |
| Space | 一時停止 |
| Esc | 状況に応じてメニューを開く、またはパネルを閉じる(GEN-010参照) |

**CTL-002** 補助操作(任意対応):矢印キー ←/→ はA/Dの代替(左右旋回)、矢印キー ↑/↓ はQ/Eの代替(航行スケール変更)。ただしパネル画面表示中は、矢印キーはパネル内の項目選択・ページ切り替えとして機能する(状態ごとに役割が変わる、CTL-005参照)。

**CTL-003** パネル(マップ/航海記録/ランキング)内操作:↑/↓は項目選択または縦スクロール、←/→は惑星系やページの切り替え、Enterは選択項目の詳細表示、PageUp/PageDownは履歴の大きなスクロール、Escはパネルを閉じて航行画面へ戻る。

**CTL-004** キー入力の扱い:

* 長押し:W/S/A/Dは押している間だけ効果が継続する(トグルではない)
* 同時押し:前進噴射しながらの旋回など、複数キーの同時入力に対応する
* キーリピート:ブラウザのデフォルトキーリピートに依存せず、`keydown`/`keyup`で押下状態を管理する
* フォーカス喪失時:ウィンドウやタブがフォーカスを失った場合、すべての押下状態を解除し、宇宙船の加速・旋回を停止する

**CTL-005** Rキー単独でゲームを初期化する仕様にはしない(誤操作防止のため)。新しい航海の開始は、確認画面を経て行う操作としてメニュー内に置く。

**CTL-009** Rキーの意味は現在表示中の画面(状態)によって変わりうる。航行中・一時停止中・マップ/航海記録/設定などの各パネル表示中は、Rキー単独でゲーム状態やセーブデータを初期化しない。ランキング画面表示中のみ、Rキーはランキングの再取得に使う(RANK-005参照)。セーブデータの初期化(消去)は、Rキーではなく設定画面内の「保存データを消去」操作と二段階確認によって行う(SAV-005参照)。

**CTL-006** Escキーの状態別動作:航行中に押すと一時停止メニューを開く。パネル表示中に押すとパネルを閉じて航行へ戻る(GEN-012の再開カウントダウンを伴う)。一時停止メニューでの「終了」系操作は、確認を経てから実行する。

**CTL-007** Tabキーによる近隣天体の巡回選択:Tabキーを押すたびに、近隣または画面内の選択可能な天体を順番に切り替える。Shift+Tabでは逆順に切り替える。ブラウザ標準のフォーカス移動と競合しないよう、ゲーム画面へフォーカスがある場合だけ天体選択として処理する。スマートフォンでは、天体一覧のタップまたは「次の天体」ボタンで同等の操作を提供する(MOB-011)。

**CTL-008** クイックセーブ・クイックロード用の専用キーは用意しない。保存は自動保存・自動再開(SAV-006)へ統一する。F5/F9はブラウザの再読み込みや開発者ツールと競合しやすく、スマートフォンに対応する物理キーもないため、正式仕様から除外する。

### 9.2 スマートフォン操作

**CTL-010** タッチ操作だけで、航行・天体選択・着陸・航行スケール変更・航海記録確認・ランキング確認・新しい航海の開始まで、すべてのゲーム機能を利用できることを必須とする(詳細は22章)。

**CTL-011** ジェスチャー対応:天体タップ(選択)、ドラッグ(視点移動/マップ移動)、ピンチイン・ピンチアウト(カメラズームのみ、航行スケールとは別扱い)、天体アイコンの横スワイプ(訪問一覧スクロール)、履歴画面の縦スワイプ(履歴スクロール)。誤操作防止のため、ダブルタップだけでの初期化・航海終了は行わない。

## 10. 通常航行画面

**UI-001** 画面中央には宇宙船と宇宙空間を大きく表示し、情報表示は画面端へ整理する。宇宙の景色を主役とし、HUDで覆い尽くさない。

**UI-002** 表示要素:宇宙船、太陽、惑星、衛星、星空(奥行きのある複数レイヤー)、控えめな星雲、軌道線、目的地方向マーカー、接近情報、現在の重力対象天体、航行状態文字列、警告表示。

**UI-003** 航行状態は次の文字列で表示する:離脱中/巡航中/接近中/周回中/降下中/着陸可能/着陸済み。

**UI-004** 天体名は常時表示せず、カーソルを合わせたとき、目的地に設定したとき、一定距離まで接近したときにのみ表示する。

**UI-005** 太陽への接近時は画面全体がわずかに明るくなり、熱による揺らぎ・警告表示を出す。ただし戦闘ゲーム風の激しいダメージ演出は入れない。

**UI-006** フォトモード:Pキー、またはスマートフォンでは画面上のカメラボタン(MOB-013)で開く。宇宙旅行と天体の景色を楽しむというゲームコンセプトに合わせた正式機能とする。フォトモード中はゲーム時間を停止する(GEN-011と同様の扱い)。フォトモードで可能な操作:

* カメラズームの調整
* HUDの表示・非表示切り替え
* 天体名の表示・非表示切り替え
* 軌道線の表示・非表示切り替え
* スクリーンショットの保存

フォトモードを閉じると、GEN-012と同様の再開カウントダウンを経て航行を再開する。

## 11. 下部操作盤(HUD)

**HUD-001** 画面下部に、半透明・濃紺基調のホログラム風パネルとして宇宙船操作盤を常時表示する。左・中央・右の3エリア構成とする。

* 左エリア:推進・旋回インジケータ。宇宙船の簡易シルエットまたは機首アイコンと、THRUST(前進噴射)/RETRO(逆噴射)/LEFT(左旋回)/RIGHT(右旋回)のランプ表示。キー入力に応じて対応ランプが点灯する
* 中央エリア:航行情報(SPD=現在速度、REL=相対速度、SCALE=航行スケール、MODE=航行状態)と得点情報(SCORE=今回の得点、BEST=自己最高得点、VISITED=今回訪問数、COMBO=連続訪問数)
* 右エリア:キー操作ガイド(現在有効な基本操作を常時表示)。「簡易表示」と「詳細表示」を切り替え可能とする

**HUD-002** 押されているキーに応じて、対応する表示を発光させる。例:Wキー押下時は「W 前進噴射」表示の発光、THRUSTランプの点灯、宇宙船後方の噴射エフェクトの増強、推力メーターの上昇、エンジン音の音量上昇を同時に行う。S/A/Dについても同様に画面表示・宇宙船演出・効果音を連動させる。

**HUD-003** 速度は数値だけでなく細い横メーターでも表示し、低速・安全速度・高速・危険速度が感覚的に分かるようにする。着陸可能な安全速度に入ると、メーター色・ラベルを控えめに強調する。

**HUD-004** 下部操作盤内に、訪問対象となる惑星・衛星の小さな画像を横一列(横スクロール可能)で表示する(天体訪問一覧、CEL-010参照)。

**HUD-005** 得点が加算された際は、操作盤付近に短い得点表示を出す(画面中央を覆う演出は行わない)。表示例:「月着陸 ＋200」「ソフトランディング ＋100」。合計得点は瞬時に切り替えず、短時間で滑らかに増加させる。自己最高得点を更新した瞬間はBEST表示を短く強調する。

**HUD-006** 操作盤には、下記のキーガイドを常時含める:H(航海記録)、L(ランキング)、Esc(戻る)、M(マップ)、Wheel(カメラズーム)、Click(天体選択)、F1(ヘルプ)。

**HUD-007** デザイン要素:半透明の濃紺パネル、背景の弱いぼかし、細い発光ライン、角を落とした形状、水色・青緑色を通常表示、琥珀色を注意表示、赤色を危険表示のみに使用、控えめなノイズ・走査線。装飾だけの無意味な数値は表示しない。表示するすべての情報にゲーム上の意味を持たせる。

## 12. 天体画像

**CEL-010** 太陽・惑星・衛星には、NASA・JPL・USGSなど利用条件が確認可能な実際の観測画像・公開素材を元にしたテクスチャを使用する。外部サイトの画像URLを無断で直接参照しない(CORS制限・配信停止・URL変更のリスクを避けるため、画像はリポジトリまたはサイト内へ保存する)。

**CEL-011** 表示は距離に応じて段階的に切り替える。

* 遠距離:小さな円形スプライト。色と明るさのみ判別可能な簡易表示
* 中距離:実画像ベースの標準テクスチャ。模様が見え始める
* 近距離:高解像度テクスチャ。大気のぼかし、雲の層、環、太陽の発光、表面の地形感を追加する

天体へ近づくにつれて画像は自然に高精細化し、切り替え時の瞬間的な点滅は起こさない。

**CEL-012** 画像加工要件:サイズ統一、明るさ・コントラスト調整、背景(黒地)の除去、円形への切り抜き、遠景用・近景用の複数サイズ作成。

**CEL-013** 配信最適化:WebPまたはAVIFを優先し、遠距離用・近距離用の画像を分離し、必要になった天体画像のみ遅延読み込みする。同じ画像を繰り返しダウンロードしない。読み込み中は代替スプライトを表示し、読み込み失敗時もゲームを停止しない(ERR-001)。

**CEL-014** 使用した天体画像の出典・利用条件は、ゲーム内クレジット画面とリポジトリ内文書(README等)へ記載する。

## 13. 着陸と接近観測

**LND-001** 固体表面を持つ天体(水星、金星、地球、月、火星とその衛星、木星・土星などの主要衛星、冥王星などの準惑星)は、宇宙船が安全速度以内で表面へ到達した場合に着陸成功とする。

**LND-002** 木星・土星・天王星・海王星には明確な固体表面がないため、通常の意味での着陸対象にはしない。これらの巨大惑星は、以下をすべて満たした場合に「接近観測成功」(または「周回観測成功」)とする。

* 指定された距離以内へ接近する
* 危険高度へ入らず一定時間航行する
* 相対速度が規定値以内である
* 接近後に安全圏へ離脱する

一覧表示・実績表示では「着陸」ではなく「接近観測」または「周回達成」と表示する。

**LND-003** 着陸時には画面を切り替えず、宇宙船が天体表面へ接近する様子を連続的に表示する。着陸成功後は天体名・到着日時・移動距離・所要時間・最大速度・着陸時相対速度を静かに表示する。着陸後は、そのまま滞在するか再び離陸するかを選べる。

**LND-004** 着陸開始条件・安全速度・衝突速度・危険大気圏突入速度は、天体ごとの設定データとして定義する(具体値は14章「調整可能パラメータ」参照)。

**LND-005** 得点の確定タイミングは、着陸成功または接近観測成功が確定した瞬間とする。同一航海中の重複訪問による得点の重複加算は行わない(SCO-006参照)。

**LND-006** 地球への着陸に関する特別条件:航海は地球付近から開始するため、開始直後に地球へ着陸しても「地球への訪問成功」とはみなさない。地球の所定の離脱圏外まで一度移動し、その後あらためて地球へ戻り、安全速度で着陸した場合にのみ地球の得点を与える。同じ航海中に得られる地球の基本得点は1回だけとする(SCO-006と同じ重複防止の考え方)。離脱圏の具体的な距離は調整可能パラメータとする(15章)。

## 14. 得点

**SCO-001** 天体ごとに、到達距離・重力・大きさ・大気・着陸可能速度などを考慮した基本得点をあらかじめ設定データとして定義する。数値は天文学的難易度の厳密な再現ではなく、ゲームとしての遊びやすさを優先して調整する。

**SCO-002** 難易度と基本得点の目安(初期値。調整可能パラメータとして扱う):

* 難易度1(入門・100〜300点):地球(100点、LND-006の特別条件を満たした場合のみ)、月、フォボス、ダイモス
* 難易度2(標準・400〜700点):水星、火星、カリスト、レア
* 難易度3(上級・800〜1,200点):金星、イオ、ガニメデ、タイタン、トリトン
* 難易度4(難関・1,500〜2,500点):エウロパ、エンケラドゥス、イアペトゥス、冥王星、その他遠距離天体
* 巨大惑星の接近観測(1,000〜2,000点):木星、土星、天王星、海王星

地球は大気を持つ天体であり、将来のバランス調整で着陸難易度・得点を変更できるよう、他の天体と同じく設定データへ分離する(CEL-001)。

**SCO-003** ソフトランディングボーナス:着陸時の相対速度が低いほど加点する。許容速度ぎりぎりは基本点のみ、安全な着陸は基本点の20%追加、非常に滑らかな着陸は基本点の50%追加。

**SCO-004** 初回到達ボーナス:プレイヤーが(過去の航海を含めて)初めて訪問した天体には基本点の50%を追加する。1天体につき一度だけ獲得できる。

**SCO-005** 連続訪問ボーナス:宇宙船を破壊せず複数の異なる天体を続けて訪問すると加算する。2天体連続=100点、3天体連続=250点、5天体連続=500点、10天体連続=1,500点。

**SCO-006** 重複訪問の扱い:同じ航海中に同じ天体へ繰り返し着陸しても、基本得点は1回の航海につき最初の到達時のみ加算する。2回目以降は着陸記録(最高評価等)のみ更新し、基本点・各種ボーナスは加算しない。

**SCO-007** 得点表示は加算理由を短く表示する(HUD-005参照)。天体ごとの基本点・ボーナス率などの数値はコードへ直接埋め込まず、設定データ(JSON等)として分離する。

**SCO-008** 航海終了時の確定処理:その航海の総得点・訪問数・訪問天体一覧・最高得点天体・最も滑らかだった着陸・航海時間・移動距離・最大速度を確定し、結果画面へ渡す。自己最高得点(ブラウザ保存)を更新した場合は「NEW PERSONAL BEST」と表示する。

## 15. 航海終了

**GEN-014** 以下の条件で宇宙船が破壊・航行不能になった場合、その航海を終了する。

* 着陸許容速度を超えて天体表面へ衝突した
* 太陽へ危険距離まで接近した
* 危険な速度で大気圏へ突入した
* その他、宇宙船が航行不能になる条件を満たした

**GEN-015** 航海終了時は短い爆発演出(宇宙船の破損、閃光、通信途絶)を表示する。派手な戦闘ゲーム風の爆発ではなく、低い衝撃音・船体振動・通信ノイズ・音声途絶を組み合わせて表現する(AUD-010参照)。

**GEN-016** 演出後、航海結果画面を表示する。表示内容:今回の総得点、自己最高得点、今回訪問した天体数、訪問した天体一覧、最も高得点だった天体、最も滑らかだった着陸、航海時間、移動距離、最大速度、世界ランキング順位(ChatGPT Sites版のみ)。

**GEN-017** 結果画面には「新しい航海を開始」「航海記録を見る」「ランキングを見る」「タイトルへ戻る」のボタンを配置する。

**GEN-018** 失敗による所持品の消失や長時間のやり直しは発生させない。次の航海はすぐに開始できる。

## 16. 航海記録画面

**UI-010** Hキーまたは画面ボタン(LOGボタン)で、全画面パネルとして航海記録画面を表示する。表示中は航行を一時停止する(GEN-011)。

**UI-011** 画面上部にプレイヤー全体の記録を表示する:自己最高得点、総航海回数、訪問済み天体数、全対象天体数、累計着陸回数、累計航行時間、累計移動距離、最も多く訪問した天体、最も難易度の高い訪問済み天体。

**UI-012** 天体一覧は惑星系ごとに分類して表示する(CEL-007の分類に準拠)。未訪問の天体は黒いシルエットで表示し、一度でも訪問した天体は実画像でカラー表示する。今回の航海で訪問した天体には明るい外枠またはチェックマークを追加表示する。

**UI-013** 天体を選択すると、天体名・種類・着陸/接近観測の区分・難易度・基本得点・初回訪問日時・最終訪問日時・訪問回数・最高獲得点・最も低かった着陸時相対速度・最短到達時間・簡単な解説を表示する。未訪問の天体は名称・難易度・基本点のみ表示し、詳しい説明を隠す設定を用意する。

**UI-014** 航海履歴は新しい順に表示する。各航海の表示項目:航海日時、最終得点、訪問天体数、訪問した天体、航海時間、最大速度、航海終了理由、自己最高記録を更新したか。航海を選択すると、訪問順と得点内訳を表示する。

**UI-015** 航海履歴はLocalStorageまたはIndexedDBへ保存する。詳細保存の上限は最新100航海とする(SAV-003参照)。自己最高記録・天体ごとの累計記録は、履歴削除後も保持する。

## 17. 世界ランキング画面

**RANK-001** Lキーまたは画面ボタン(RANKボタン)で、全画面パネルとして世界ランキング画面を表示する。ChatGPT Sites版ではD1データベースから世界ランキングを取得する。表示中は航行を一時停止する(GEN-011)。

**RANK-002** 表示項目:順位、公開用プレイヤー名、自己最高得点、訪問天体数、記録日時。上位10名を表示する。

**RANK-003** 同点の場合の順位決定順:(1)訪問天体数が多い、(2)航海時間が短い、(3)記録日時が早い。

**RANK-004** ランキングには各ユーザーの最高得点のみを登録する。自己最高得点を更新した場合は登録内容を更新する。ログイン中のプレイヤーが上位10名に入っていない場合は、下部に本人の順位を別枠で表示する。

**RANK-005** ランキング取得中は読み込み表示を出す。通信失敗時はゲームを停止せず「ランキングを取得できませんでした。Escキーで航行へ戻れます」と表示し、再読み込みボタンおよびRキーによる再取得に対応する(Rキーの挙動の全体方針はCTL-009を参照、GitHub Pages単体では本画面自体を提供しない、DEP-002参照)。

**RANK-006** ランキング画面は「宇宙管制センターの通信記録」を思わせるデザインとし、上位3名のみ控えめに強調する(1位も金色装飾ではなく白または淡い琥珀色の発光にとどめる)。

## 18. ブラウザ保存

**SAV-001** 以下をLocalStorageまたはIndexedDBへ保存する:自己最高得点、訪問済み天体、天体ごとの最高記録(最高着陸評価・訪問回数・初回訪問日時・最終訪問日時)、過去の航海履歴、進行中の航海、操作設定、音量設定、表示設定、プレイヤー名、未送信スコア。

**SAV-002** 保存データにはバージョン番号を持たせ、将来のスキーマ変更に備えた移行処理を可能にする。

**SAV-003** 航海履歴の詳細保存は最新100航海を上限とする。自己最高記録・天体ごとの累計記録は履歴削除後も保持する。

**SAV-004** 保存データが存在しない場合、または破損している場合でもゲーム全体を停止せず、安全な初期状態へ復旧し、新しい航海を開始できるようにする(ERR-005)。

**SAV-005** 設定画面にローカル保存データを消去する機能を用意する。消去操作には二段階確認を設ける(誤操作防止、CTL-005と同様の思想)。

**SAV-006** ブラウザを閉じても、次回アクセス時に進行中の航海を再開できるようにする。手動でのクイックセーブ・クイックロード操作(専用キー)は用意せず、自動保存へ統一する(CTL-008)。自動保存は以下のタイミングで実行する。

* 天体訪問成功時(着陸・接近観測の確定時)
* 得点加算時
* 設定変更時
* 一時停止時
* 航海記録画面を開いた時
* ページが非表示になる時(タブ切り替え・バックグラウンド化)
* 一定時間ごと(周期的な自動保存)

## 19. D1データベース(ChatGPT Sites版)

**DB-001** ChatGPT Sites版ではD1リレーショナルデータベースを使用し、ランキングに必要な最小限の情報のみを保存する。

**DB-002** スコア記録テーブルの項目例:score_id(主キー)、player_id、display_name、score、visited_count、visited_bodies、voyage_seconds、maximum_speed、achieved_at、game_version、verification_status。

**DB-003** プレイヤー情報テーブルの項目例:player_id(主キー)、display_name、created_at、updated_at、best_score。

**DB-004** メールアドレスなど個人情報はランキング画面へ表示しない。表示するのはプレイヤー本人が設定した公開用ニックネームのみとする。

**DB-005** インデックス:scoreの降順取得、player_idによる本人記録取得を高速化するインデックスを設ける。

**DB-006** 更新条件:各ユーザーについて、ランキングにはそのユーザーの最高記録のみを反映する。新しい自己最高得点が確定した場合のみレコードを更新する。

**DB-007** 上位10名取得・本人順位取得の2種類のクエリを提供する。

## 20. ユーザー認証

**SEC-001** ゲームは未ログインでもプレイ可能とする。ローカルの自己最高得点・訪問履歴・航海履歴は、未ログインでもブラウザへ保存する。

**SEC-002** 世界ランキングへの登録時だけ認証を求める。ChatGPT Sitesが提供するサイト内認証機能を使用してランキング登録者を識別する。利用可能な場合は「Sign in with ChatGPT」を使用する。具体的な認証API呼び出し方法、取得できるユーザー情報、ユーザー識別子の形式は、本仕様書では特定の未確認APIを前提に固定しない。実装時点で対象環境(ChatGPT Sites)に提供されている正式な仕様に従う。

**SEC-003** 認証機能が利用できない場合もゲーム本体は動作させる。認証できない場合は、ランキングの閲覧のみ許可し、登録は行わせない。

**SEC-004** ランキングへ表示する名前は、ユーザーが設定した公開用ニックネームとする。ChatGPTの氏名やメールアドレスをランキングへ直接表示しない。認証ユーザーIDはサーバー側だけで扱い、クライアントへ露出させない。

**SEC-005** 最初のランキング登録時に、公開用のプレイヤー名を入力させる。プレイヤー名の検証条件:2文字以上16文字以下、制御文字禁止、HTMLタグ無効化(エスケープ)、不適切な名称を拒否できる構成、同一ユーザーによる短時間の連続変更を制限する。

**SEC-006** ユーザーを識別する処理とスコア登録処理は、必ずサーバー側(ChatGPT Sites側の処理)で行う。クライアント側の値だけで認証・登録を完了させない。

---

## 21. 不正スコア対策

**SEC-010** クライアントから送られてきた得点をそのまま信用して登録しない。最低限、サーバー側で以下を検証する:得点が整数であること、得点が0以上であること、訪問天体と得点の組み合わせに矛盾がないこと、同じ天体の得点が重複加算されていないこと、航海時間に対して到達天体数が不自然でないこと、ゲームのバージョンが有効であること、同一ユーザーからの短時間大量登録がないこと。

**SEC-011** 可能な範囲で、天体訪問時にサーバーへ航海イベントを記録し、航海終了時にサーバー側で合計点を再計算する。クライアントが送信する最終得点だけに依存しない構成とする。

**SEC-012** 各航海には一意なIDを割り当て、同一航海IDによる重複登録を防止する。

**SEC-013** 明らかに不正と判断した記録はランキングへ表示しない。

**SEC-014** その他のセキュリティ要件:SQLインジェクション対策としてパラメータ化クエリを用いる、エラーメッセージで内部情報(スタックトレース、テーブル名など)を公開しない、レート制限を設ける。

## 22. オフライン時・通信失敗時

**ERR-001** D1への接続に失敗しても、ゲーム自体は続行できるようにする。通信失敗時は自己最高得点・航海記録をブラウザへ保存し、「ローカル記録として保存しました」と表示する。

**ERR-002** 同じスコアが何度も送信されないよう、各航海に一意なIDを割り当てる(SEC-012と同一の仕組み)。通信が復旧した場合、未送信記録を再送できるようにする。

## 23. UIデザイン方針

**UI-020** デザインコンセプトは「未来の戦闘機」ではなく「静かな民間宇宙旅行船・天体観測船」とする。使用要素:深宇宙の背景(濃紺〜黒のグラデーション)、複数レイヤーの奥行きのある星空、半透明パネル、背景ぼかし、発光ライン、ホログラム風表示、太陽の発光、惑星の大気、控えめな粒子、滑らかな画面遷移、押下中キーの発光、未訪問天体のカラー化演出。

**UI-021** 色は黒・濃紺・白を基本とし、情報表示には淡い水色・琥珀色を使う。危険表示のみ赤色を使用する(赤色の常用は禁止、25章)。

**UI-022** 起動画面:暗い画面に星が現れる→太陽系の軌道線が薄く描画される→タイトル「SPACE TRAVEL」表示→副題表示→地球と月が現れる→「宇宙旅行を開始」ボタン表示、という流れの起動演出を行う。起動画面のメニュー項目:宇宙旅行を開始、続きから、航海記録、世界ランキング、操作方法、設定、クレジット。新しい航海を開始するまで宇宙船は移動させない。

**UI-023** 画面遷移(マップ/航海記録/ランキング/設定を開閉する際)は、背景を少し暗くする→パネルが滑らかに現れる→見出しがフェードインする→内容が段階的に表示される、という流れとする。画面遷移中に複数回キーを押してもパネルが重複表示されないようにする。

**UI-024** 天体訪問一覧の演出:未訪問(黒いシルエット、名称は薄く表示)→初回到達時に光が走る→天体画像に色が付く→外周が一度発光する→天体名と得点が表示される→チェックマークが付く、という一連の演出を行う。演出中もゲームを長時間停止させない(GEN-013)。

**UI-025** 初回読み込み中は、無地のローディング画面ではなく「NAVIGATION SYSTEM」「CELESTIAL DATABASE」「FLIGHT CONTROL」「AUDIO SYSTEM」「LOCAL RECORDS」「RANKING LINK」のような宇宙船システム起動を思わせる表示を行う。実際には読み込んでいない項目を偽って長時間表示しない。初回読み込みはゲーム本体・UI・地球・月・太陽・宇宙船・基本効果音に限定し、遠方の惑星・高解像度画像・記録用画像・追加効果音は遅延読み込みとする(PERF-006)。

## 24. 効果音とBGM

**AUD-001** 効果音を必ず実装する。宇宙空間そのものの外部音ではなく、宇宙船内で聞こえる機器音・船体振動・操作音として表現する。方向性:落ち着いている、低音中心、長時間遊んでも疲れにくい、未来的だが耳障りでない、攻撃的でない、操作状態が音だけでも分かる。

**AUD-002** 最低限用意する効果音:

* UI操作:選択音、決定音、戻る音、メニュー音、パネル閉じる音、タブ切り替え音、設定変更音、エラー音
* 宇宙船操作:エンジン始動音、前進噴射ループ音、逆噴射ループ音、旋回スラスター音、航行スケール変更音、カメラズーム音、速度警告音、接近警告音、着陸可能通知音
* 天体・航海:天体選択音、新規発見音、初回訪問達成音、着陸成功音、接近観測成功音、得点加算音、連続訪問ボーナス音、自己最高記録更新音
* 航海終了:危険警告音、衝突音、船体損傷音、爆発/通信途絶音、航海結果表示音

**AUD-010** エンジン音は、押している間だけ自然に継続するループ音とし、キー入力のたびに短音を繰り返す方式にはしない。Wキー押下時は音量を短時間で上げてループし、キーを離すと滑らかに音量を下げ、推力に応じて音の高さをわずかに変える。逆噴射・旋回音は前進噴射音と区別できる音にする。爆発音は低い衝撃音・船体振動・通信ノイズ・音声途絶の組み合わせとし、戦闘ゲーム風の爆発音にはしない。

**AUD-011** 音声ファイルは複数ブラウザ対応形式(WebMまたはOpus、MP3の代替ファイル)で用意する。短い効果音は音声スプライトの利用を検討し、エンジン音・BGMなど長時間ループする音は個別ファイルとする。ファイル名は用途が分かるようにする(例:`engine-thrust-loop.webm`)。

**AUD-020** スマートフォンでは、ページを開いただけで音声を自動再生しない。「宇宙旅行を開始」ボタンなど最初のユーザー操作を受けた後に音声機能を有効化する。音声を開始できなくてもゲーム本体は正常に開始し、設定画面に状態を表示する。

**AUD-021** 音量設定項目:マスター音量、効果音音量、エンジン音量、UI音量、BGM音量、警告音量、ミュート、振動の有効・無効。設定値はブラウザへ保存する。初回起動時の音量は大きすぎない値にする。警告音を無効にした場合は、画面上の警告表示を明確にする(視覚的フォールバック、UI-030参照)。

**AUD-030** BGMは静かなアンビエント音楽とし、場面(タイトル/通常航行/接近/着陸準備/航海終了/航海記録)ごとに変化させてもよいが、曲数よりも長時間聴いても疲れないことを優先する。利用許諾が明確な素材またはオリジナル音源のみを使用する。BGMは完全に無効化できるようにする。

## 25. アクセシビリティ

**UI-030** 「演出を減らす」設定を用意し、有効時は大きな画面移動、点滅、パーティクル、背景の視差、強いズーム、長い画面遷移、継続的なHUDアニメーションを抑制する。

**UI-031** 音を出せない環境でも遊べるよう、重要な効果音には必ず視覚表示を併用する(例:警告音と警告パネル、着陸可能音と「LANDING READY」表示、得点音と加算点表示)。色だけで状態を区別せず、文字・アイコン・形状も併用する。

## 26. スマートフォンUI

**MOB-001** スマートフォン版は、PC版の操作盤をそのまま縮小した構成にせず、専用レイアウトを設計する。横画面で左右の親指により操作できる配置とする。

**MOB-002** 左側タッチパッド:前進噴射、逆噴射、左旋回、右旋回。ボタンは押している間だけ操作を継続し、複数ボタンの同時押下(前進しながら旋回等)に対応する。

**MOB-003** 右側操作:航行スケール拡大(＋)、航行スケール縮小(－)、MAP、LOG(航海記録)、RANK(世界ランキング)、MENU(一時停止メニュー)。ボタンにはアイコンと短いラベル(例:「THRUST」「W」の併記)を表示し、PC操作との対応関係を分かりやすくする。

**MOB-004** タッチボタンは半透明とし、押下時に発光・わずかな縮小・短い効果音・(対応端末では)弱い振動・操作盤インジケーターの点灯を返す。

**MOB-005** 下部操作盤は「常時表示」と「折りたたみ表示」に分ける。

* 常時表示:今回の得点、自己最高得点、現在速度、相対速度、選択中の天体、航行スケール、着陸可能表示、前進/逆噴射/左右旋回ボタン
* 折りたたみ表示(上へスワイプで展開):今回訪問した天体、未訪問天体一覧、得点の内訳、詳細な航行情報、操作説明、航海時間、最大速度

**MOB-006** 天体訪問一覧は画面最下部へ横スクロール形式で表示する。天体アイコンは指で確実に選択できる大きさを確保し、短いタップで目的地設定、長押しで詳細表示とする。マウスホバーに依存する情報表示は用意しない。

**MOB-007** 航海記録・世界ランキングは、H/Lキーに相当するLOGボタン・RANKボタンで開く。Androidの戻る操作・ブラウザの戻る操作でもゲームを終了せず、一つ前のゲーム内画面へ戻す。パネル表示中はPC版同様にゲーム時間を一時停止し、閉じた後は再開カウントダウンを表示する(GEN-011、GEN-012と共通)。

**MOB-008** タッチ操作の設定項目:ボタンの大きさ、ボタンの透明度、左右配置の交換(左利き対応)、振動フィードバックの有効・無効、誤操作防止の感度、ピンチズームの感度、操作盤の表示位置。設定値はブラウザへ保存する。

**MOB-009** 誤操作防止策:操作ボタン付近のページスクロール抑制、長押しによる文字選択の禁止、ダブルタップによるページ拡大の抑制、操縦中の意図しないブラウザスクロールの防止、メニュー操作中は宇宙船を動かさない、新しい航海の開始・保存データ消去には確認画面/二段階確認を設ける。

**MOB-010** 振動フィードバック(対応端末のみ):ボタン押下時は非常に短い振動、着陸成功時は短い振動2回、危険速度時は弱い警告振動、衝突・爆発時はやや長い振動。振動非対応環境でも正常動作すること。初期設定は弱い振動または無効とする。

**MOB-011** Tabキーに相当する近隣天体の巡回選択(CTL-007)は、スマートフォンでは天体一覧のタップ、または操作盤内の「次の天体」ボタンで提供する。

**MOB-012** 表示モード切り替え(C、PHY-004)は、スマートフォンでは設定画面、または操作盤内のカメラ設定から変更できるようにする。

**MOB-013** フォトモード(P、UI-006)は、スマートフォンでは画面上のカメラボタンから開けるようにする。

## 27. 性能

**PERF-001** 目標フレームレート:PC=60fps、一般的なスマートフォン=30fps以上、高性能スマートフォン=60fps目標。

**PERF-002** 端末性能に応じて調整可能な項目:星の数、粒子の数、発光エフェクト、ぼかし、影、天体画像解像度、アニメーション頻度、太陽コロナの品質、軌道予測線の精度、背景エフェクト、爆発エフェクト。

**PERF-003** 表示品質は「高画質」「標準」「軽量」「自動」から選択できるようにする。「自動」では初回起動時または実行中の実測フレームレートを参考に品質を調整する。低性能端末向けの「軽量表示モード」を用意する。

**PERF-004** モバイル回線でも初回読み込みが過度に長くならないよう、素材を初回読み込み分(UI-025参照)と遅延読み込み分に分割する。

## 28. エラー処理

**ERR-005** 以下の障害が発生してもゲーム全体を停止しない:天体画像の読み込み失敗、音声の読み込み失敗、CDNの読み込み失敗、LocalStorage/IndexedDB利用不可、D1接続失敗、ランキング取得失敗、スコア送信失敗、保存データ破損。

**ERR-006** 各障害への対応方針:

* 画像読み込み失敗:代替スプライトまたは簡易表示に切り替える
* 音声読み込み失敗:無音のまま進行し、視覚表示(UI-031)で代替する
* CDN読み込み失敗:ENV-014のフォールバックに従う
* ブラウザ保存不可:その場のメモリ上でのみ状態を保持し、警告なく通常プレイを継続する(保存されない旨は設定画面に表示する)
* D1接続失敗・ランキング取得失敗・スコア送信失敗:ERR-001/ERR-002に従いローカル保存・再送を行う
* 保存データ破損:安全な初期状態へ復旧し、新しい航海を開始できるようにする(SAV-004)

## 31. 受け入れ条件(要件ID対応表)

以下、要件IDと受け入れ条件の対応例。実装時はテスト項目としてそのまま利用する。

### CTL-001/HUD-002 基本操作とキー発光の連動

要件:PCでW/S/A/Dを押すと宇宙船が加速・旋回し、下部操作盤の対応表示が発光する。

受け入れ条件:
* Wキー押下中、宇宙船が前進方向へ加速する
* Wキー押下中、操作盤のTHRUSTランプと「W 前進噴射」表示が発光する
* キーを離すと加速と発光が止まる
* 複数キー同時押下(例:W+A)で前進しながら旋回できる

### LND-002 巨大惑星の接近観測

要件:木星・土星・天王星・海王星は着陸させず、接近観測条件の達成で「接近観測成功」とする。

受け入れ条件:
* 木星への安全な接近・離脱で「接近観測成功」と表示され、得点が加算される
* 木星の一覧表示に「着陸」の文字が使われていない
* 危険高度への進入や超過速度での接近は観測失敗として扱われる

### SCO-006 重複訪問時の得点

要件:同一航海中に同じ天体へ複数回着陸しても、基本得点は最初の1回のみ加算する。

受け入れ条件:
* 月へ1回目の着陸で基本得点が加算される
* 同じ航海中に月へ2回目の着陸をしても基本得点は加算されない
* 2回目の着陸でも着陸評価記録(最高速度等)は更新される

### CEL-010/UI-024 天体訪問表示

要件:未訪問の天体は黒いシルエットで表示し、着陸または接近観測成功後は実画像によるカラー表示へ切り替える。

受け入れ条件:
* 新規データでは対象天体が黒いシルエットになる
* 月への初回着陸後、月がカラー表示になる
* 得点が加算される
* 初回訪問日時が保存される
* ブラウザ再起動後もカラー表示が維持される
* スマートフォンでも同じ状態が表示される

### GEN-011/GEN-012 パネル表示中の一時停止と再開

要件:マップ・航海記録・ランキング・設定パネルの表示中は物理演算を停止し、閉じた際はカウントダウンを経て再開する。

受け入れ条件:
* Hキーで航海記録パネルを開くと、宇宙船の移動と重力計算が停止する
* パネル表示中に航海時間が加算されない
* Escキーでパネルを閉じると「3・2・1・航行再開」のカウントダウンが表示される
* カウントダウン中はW/S/A/Dの入力が反映されない
* カウントダウン終了後、通常操作が再開する

### RANK-001/ERR-001 ランキング通信失敗時の継続動作

要件:D1への接続に失敗してもゲーム進行を止めない。

受け入れ条件:
* ネットワーク遮断状態でLキーを押しても、ランキング画面がクラッシュせず「取得できませんでした」表示が出る
* Escキーで航行画面へ戻れる
* 自己最高得点・航海記録はローカルへ保存される
* 通信復旧後、未送信スコアが再送される

### MOB-001〜MOB-010 スマートフォン操作の完結性

要件:キーボードなしですべての操作を完結できる。

受け入れ条件:
* キーボードなしで新しい航海を開始できる
* タッチ操作だけで宇宙船を加速、減速、旋回できる
* タッチ操作だけで航行スケールを変更できる
* 天体をタップして目的地に設定できる
* 惑星または衛星へ着陸できる
* 天体訪問一覧を横スクロールできる
* 航海記録・世界ランキングを表示して航行画面へ戻れる
* 画面回転後もゲーム状態が失われない
* Android Chrome、iPhone Safariで正常に操作できる

### ENV-013/ENV-014 CDNバージョン固定とフォールバック

要件:CDNライブラリは`latest`を指定せず、失敗時は代替表示に切り替える。

受け入れ条件:
* package.jsonまたはHTML内のCDN読み込みURLにすべて明示的なバージョン番号がある
* `latest`という文字列がCDN参照に含まれない
* GSAPの読み込みを意図的に失敗させても、画面遷移がCSSアニメーションで機能する
* Three.jsの読み込みを意図的に失敗させても、背景がCanvas 2Dで描画される

## 33. 調整可能パラメータ

以下は数値をコードへ固定せず、設定データとして持たせ、実プレイのバランス調整で変更することを前提とするパラメータ。初期値は`memo.md`の記載または類推値であり、確定値ではない。

* 巨大惑星の接近観測成功距離・安全滞在時間・相対速度上限
* 太陽への危険接近距離
* 地球の離脱圏外判定距離(LND-006、航海開始直後の即着陸を無効化する条件)

* 航海履歴の詳細保存上限件数(初期値100航海)
* 音量の初期値(マスター・効果音・エンジン・UI・BGM・警告の各初期音量)
* 画質自動判定のフレームレート基準値(PERF-003の「自動」モード)
* プレイヤー名の文字数制限(初期値2〜16文字)
