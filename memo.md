# スペース・トラベル (Space Travel) 仕様メモ

出典: [Wikipedia日本語版「スペース・トラベル (ビデオゲーム)」](https://ja.wikipedia.org/wiki/%E3%82%B9%E3%83%9A%E3%83%BC%E3%82%B9%E3%83%BB%E3%83%88%E3%83%A9%E3%83%99%E3%83%AB_(%E3%83%93%E3%83%87%E3%82%AA%E3%82%B2%E3%83%BC%E3%83%A0))

## 基本情報

| 項目 | 内容 |
|---|---|
| 開発者 | ケン・トンプソン |
| 開発元 | ベル研究所 |
| 発売日 | 1969年 |
| ジャンル | シミュレーションゲーム |
| 対応機種 | Multics、GECOS (GE 635用)、PDP-7 |
| プレイ人数 | シングルプレイヤー |

## ゲームシステム

太陽系の旅をシミュレーションするゲームで、プレイヤーは2次元空間上で宇宙船を操作する。

- **視点**: 真上から見下ろす2次元平面
- **表示**: モノクロ表示。黒い背景に白い線でオブジェクトを描画
- **操作**: 前進・後進・方向転換が可能。画面の拡大縮小によって速度を調整する
- **目的**: 惑星や衛星への着陸のみ(それ以外の明確なゴールはない)
- **重力システム**: 宇宙船の周辺で最も強い引力を持つ天体1つだけが影響を与える(簡易的な重力モデル)

## 開発経緯

- トンプソンはもともとGE 635上のMultics用に書かれた「スペース・トラベル」をFORTRANに移植して遊んでいた。
- GE 635版の問題点:
  - 1回のプレイに$50〜$75ものコストがかかった
  - 他のジョブが実行中だと長時間待たされた
  - リアルタイムのボタン入力に対応していなかった
- そこでトンプソンは、空いていたPDP-7へ移植することを決意し、ゲームを一から書き直した。
- この過程で「ライブラリから」開発を行うことになり、アセンブラを自作。さらにデニス・リッチーらと協力して独自のファイルシステムを構築した。

## 歴史的位置づけ

- ビデオゲーム市場が立ち上がる前、初期のメインフレーム上で作られたゲームであり、ベル研究所の外部に配布されることはなかった。
- ゲーム自体がビデオゲーム史に与えた影響はほぼない。
- 最大の歴史的重要性は、このゲームの移植・開発作業がきっかけとなり、1970年に**UNIX**という基本的ながら完全なOSが誕生したことにある。

## オリジナル仕様(原典より・補足)

日本語版Wikipediaの記述より詳細な、Dennis Ritchie本人の解説記事「Space Travel: Exploring the solar system and the PDP-7」および英語版Wikipediaに基づく原典仕様。

### 操作(オリジナルのハードウェアスイッチ)
- 前進・後退のジェット噴射用スイッチ
- 左右回転用スイッチ
- 表示スケールの拡大・縮小用スイッチ
- GECOS(GE 635)版はボタンではなくコマンドをタイプする方式で、操作性が悪かった
- 加速度は「ピクセル/秒²」で固定値だが、表示スケールに応じて見かけの動きが変わる

### 座標系・表示
- 宇宙船は常に画面中央に、上向き固定で表示される
- 回転操作は宇宙船ではなく太陽系全体(座標系)を回転させるもの
- オプション機能として、表示座標系を「最も近い天体」を基準に回転させるモードも実装されていた
- 計算上は、宇宙船に最も強く重力を及ぼす天体を原点とする座標系を使い、数値の安定性を保っていた

### 物理・ゲームルール
- 惑星・衛星(大半)は軌道半径・質量とも実際のスケールに比例して再現されているが、軌道は円軌道に簡略化
- 天体は平面上をスケールされた速度で周回運動する
- 重力は逆二乗則で計算されるが、宇宙船に作用するのは「その時点で最も強い引力を持つ天体1つ」のみ(合成しない)
- 着陸(Land)の定義:天体表面のラインを、十分に低い速度で通過すること
- スケールを大きくすると太陽系を数秒で横断できるが、目標を行き過ぎる(オーバーシュートする)リスクがある

### 実装の技術的詳細
- 全編PDP-7アセンブリ言語で記述
- 完全な浮動小数点演算シミュレータを自前で実装
- 組み込みデバッガを搭載
- PDP-7移植時にライブラリ・アセンブラを新規に自作する必要があり、これがのちのファイルシステム/OS(UNIX)開発につながった

## キーバインド(現代PCゲーム向け再現案)

オリジナルのPDP-7版は専用ハードウェアスイッチによる操作だったため、現代のPC向けに再現する場合はWASD周辺にまとめるのが操作しやすい。

### 基本操作

| キー | 動作 |
|---|---|
| W | 押し上げる |
| S | 押し下げる |
| A | 左にヨー |
| D | 右にヨー |
| E | スケールを拡大 |
| Q | スケールを縮小 |

### 補助操作

| キー | 動作 |
|---|---|
| ← / → | 左・右にヨー |
| ↑ / ↓ | スケールを拡大・縮小 |
| マウスホイール上／下 | スケールを拡大・縮小 |

### ゲーム操作

| キー | 動作 |
|---|---|
| R | 新しいゲーム／ゲームをやり直す |
| Esc | 終了メニューを開く |
| Enter | 終了や新規ゲームの確認 |
| F1 | 操作説明を表示 |

**注意**: Escは押しただけで即終了させず、終了確認メニューを挟むこと。操作ミスによる不意のゲーム終了(宇宙旅行の突然の中断)を防ぐため。

```

```

# スペース・トラベル 現代版アレンジ仕様案

## 1. ゲームのコンセプト

太陽系を自由に旅し、惑星や衛星を訪問する宇宙旅行シミュレーション。

敵、戦闘、武器、資源採掘、国家間競争などは登場させない。

プレイヤーは宇宙船を操縦し、天体の重力を利用しながら目的地へ向かう。高速航行時の雄大さと、着陸時の繊細な操作の両方を楽しむゲームとする。

明確なクリア条件は設けず、プレイヤー自身が次の目的地を決める。

主な楽しみは次のとおり。

* 太陽系を自由に移動する
* 惑星や衛星へ接近する
* 軌道や重力の変化を観察する
* 安全に着陸する
* 訪れた天体を旅行記録に残す
* 宇宙の景色を眺める

## 2. 基本システム

オリジナルの特徴を残し、宇宙船は基本的に画面中央へ表示する。

宇宙船の向きを変える操作では、宇宙船そのものを大きく回転表示するのではなく、周囲の太陽系が回転する表現を採用する。

ただし、現代のプレイヤーが混乱しないように、次の表示モードを選択できるようにする。

### 船体固定モード

宇宙船は画面中央で上向きに固定され、周囲の天体と軌道が回転する。

オリジナルに近い表示方式。

### 方角固定モード

宇宙空間の方角を固定し、宇宙船が画面上で回転する。

一般的な現代ゲームに近い表示方式。

初期設定は船体固定モードとする。

## 3. 現代的なグラフィック

ゲーム画面は2Dの見下ろし型を維持しながら、光、粒子、奥行き表現を加える。

完全な3Dゲームにはせず、操作の分かりやすさとオリジナルの雰囲気を優先する。

### 宇宙空間

背景には暗い星空を表示する。

星はすべて同じ速度で流さず、複数の奥行きに分けてわずかに動かす。これにより、2D画面でも移動している感覚を出す。

星雲や銀河は控えめに表示し、画面が派手になりすぎないようにする。

### 太陽

太陽には柔らかな発光、コロナ、わずかな表面変化を加える。

近づくほど画面全体が明るくなり、警告表示や熱による揺らぎが現れる。

ただし、宇宙旅行を妨げる激しいダメージ演出は入れない。

### 惑星と衛星

各天体は色、大気、模様、輪などによって識別できるようにする。

遠距離では小さな光点として表示し、接近すると徐々に表面模様が見えるようにする。

天体名は常時表示せず、カーソルを合わせたとき、目的地に設定したとき、一定距離まで接近したときに表示する。

### 宇宙船

宇宙船は小型でシンプルなシルエットとする。

噴射時にはエンジンの光と粒子を表示する。

噴射の強さに応じて光の長さを変え、現在加速している方向を視覚的に分かるようにする。

宇宙船を過度に大きく表示せず、宇宙の広さを感じられる比率にする。

## 4. 画面構成

画面中央には宇宙船と宇宙空間を大きく表示する。

情報表示は画面の端へ整理し、宇宙の景色を邪魔しないようにする。

### 左上

現在地に関する情報を表示する。

* 最も強い重力を及ぼしている天体
* その天体までの距離
* 相対速度
* 現在の航行状態

航行状態は次のような言葉で表示する。

* 離脱中
* 巡航中
* 接近中
* 周回中
* 降下中
* 着陸可能
* 着陸済み

### 右上

選択中の目的地を表示する。

* 目的地名
* 目的地までの距離
* 推定接近時間
* 目的地との相対速度
* 接近方向

目的地を選択していない場合は「自由航行」と表示する。

### 画面下部

宇宙船の速度と噴射状態を、細いメーターで表示する。

数字だけでなく、低速、安全速度、高速、危険速度が感覚的に分かるようにする。

着陸可能な速度まで減速すると、メーターが静かに強調表示される。

### 中央付近

常時表示する照準器は置かない。

目的地を選択した場合のみ、画面端に目的地の方向を示す小さなマーカーを表示する。

画面外の天体に対しても、どちらへ旋回すればよいか分かるようにする。

## 5. 航行スケール

オリジナルのスケール変更は、現代版では「航行スケール」として明確に表示する。

航行スケールは単なる画面拡大ではなく、宇宙船をどの距離単位で操作しているかを示す。

次の段階を用意する。

* 着陸
* 接近
* 惑星圏
* 惑星間
* 太陽系

着陸スケールでは細かな速度調整がしやすい。

太陽系スケールでは長距離を短時間で移動できるが、目的地を通過しやすくなる。

スケール変更時には画面を急に切り替えず、滑らかにズームさせる。

現在の航行スケールは画面下部に短く表示する。

## 6. カメラズームとの分離

航行スケールとカメラのズームは別の機能として扱う。

* Q／E：航行スケールの変更
* マウスホイール：カメラ表示の拡大・縮小

これにより、「速度を調整したいのに画面だけ拡大された」「景色を見たいだけなのに航行速度が変わった」という混乱を防ぐ。

カメラズームは一定時間操作しなければ、現在の航行スケールに適した倍率へゆっくり戻してもよい。

## 7. 操作方法

### 基本操作

* W：前進噴射
* S：逆噴射
* A：左旋回
* D：右旋回
* Q：航行スケールを狭くする
* E：航行スケールを広くする

### カメラと情報

* マウスホイール：カメラズーム
* マウス左クリック：天体を選択
* Tab：近くの天体を順番に選択
* M：太陽系マップ
* C：表示モード切り替え
* F1：操作説明

### ゲーム操作

* Space：一時停止
* Esc：メニュー
* F5：クイックセーブ
* F9：クイックロード

新しい旅行を開始する操作は、誤操作を防ぐためメニュー内に置く。

Rキー単独でゲームを初期化する仕様にはしない。

## 8. 太陽系マップ

Mキーで太陽系全体を確認できるマップを開く。

マップには次の情報を表示する。

* 太陽
* 惑星
* 主な衛星
* 軌道
* 宇宙船の現在位置
* 宇宙船の進行方向
* 過去の航跡
* 選択中の目的地

天体を選択すると、簡単な説明と訪問記録を表示する。

マップ画面から目的地を設定できるが、完全な自動操縦は行わない。

目的地への方向は示すが、実際の旋回、加速、減速、着陸はプレイヤーが操作する。

## 9. 航行支援

操作に慣れていないプレイヤー向けに、任意で航行支援を有効にできる。

### 軌道予測線

現在の速度と重力から、宇宙船がこのまま進んだ場合のおおよその進路を表示する。

予測線は完全な未来を保証するものではなく、噴射や重力対象の変化によって随時更新する。

### 接近予測

目的地へ最も接近する予測地点を表示する。

目的地を大きく通過しそうな場合は「通過予定」と表示する。

### 着陸支援

着陸可能な速度と進入方向を表示する。

速度が高すぎる場合は「減速してください」と表示する。

着陸可能な状態になると、天体表面の安全な進入範囲を淡く表示する。

### 姿勢安定

キーを離したときに旋回をゆっくり止める補助機能を選択できる。

オリジナルに近い慣性操作を楽しみたい場合は無効にできる。

## 10. 着陸演出

十分に低い相対速度で天体表面へ到達すると着陸成功とする。

着陸時には画面を切り替えず、宇宙船が天体表面へ接近していく様子を連続的に表示する。

着陸成功後は、次の情報を静かに表示する。

* 天体名
* 到着日時
* 出発地
* 移動距離
* 所要時間
* 最大速度
* 着陸時の相対速度

派手な得点表示や勝利演出は行わない。

「到着しました」という短い表示と、落ち着いた音だけにする。

着陸後は、そのまま滞在するか、再び離陸するかを選べる。

## 11. 宇宙旅行記録

訪問した天体は「航海日誌」に記録する。

航海日誌には次の内容を保存する。

* 訪問した天体
* 初回到着日時
* 着陸回数
* 最短到着時間
* 最も安全だった着陸
* プレイヤーが撮影した画像
* プレイヤーが入力した短いメモ

ゲームの進行度は得点ではなく、訪問記録として表現する。

すべての天体を訪問してもゲームは終了しない。

## 12. フォトモード

Pキーでフォトモードを開く。

フォトモード中はゲーム時間を停止し、画面上の情報を非表示にできる。

次の調整を可能にする。

* カメラ倍率
* 宇宙船の表示位置
* 天体名の表示
* 軌道線の表示
* 星の明るさ
* 画像保存

宇宙旅行を楽しむゲームなので、フォトモードは主要機能として扱う。

## 13. 音響

宇宙空間そのものには大きな環境音を付けない。

代わりに宇宙船内部で聞こえる音として表現する。

* エンジンの低い振動音
* 操作スイッチ音
* 航行スケール変更音
* 目的地接近通知
* 着陸成功音
* 船内機器の静かな作動音

BGMは控えめなアンビエント音楽とする。

長時間遊んでも疲れにくい音量と曲調にし、BGMを完全に無効化できるようにする。

## 14. ゲームモード

### エクスプローラーモード

現代版の標準モード。

航行支援、軌道予測、目的地方向表示を利用できる。

### クラシックモード

オリジナルに近い表示と操作を楽しむ。

モノクロ線画表示、船体固定、簡略HUD、最強重力天体のみの物理モデルを使用する。

### リラックスモード

着陸可能速度の範囲を広げ、衝突時も直前から再開できる。

宇宙旅行と景色を気軽に楽しむためのモード。

## 15. 失敗時の扱い

高速で天体へ衝突した場合でも、爆発を派手に見せない。

画面を暗転させ、「着陸速度を超過しました」と表示する。

次の選択肢を出す。

* 接近開始時点から再開
* 最後のセーブから再開
* 自由航行へ戻る

失敗による所持品の消失や長時間のやり直しは発生させない。

## 16. 初回チュートリアル

最初の目的地は月とする。

チュートリアルでは次の順に案内する。

1. 前進噴射
2. 左右旋回
3. 航行スケール変更
4. 月を目的地に設定
5. 軌道予測線を確認
6. 月へ接近
7. 逆噴射で減速
8. 月面へ着陸

説明は画面を埋める文章ではなく、必要な場面で一つずつ表示する。

チュートリアル終了後は、地球から再出発しても、そのまま月から旅を続けてもよい。

## 17. デザイン方針

画面は「未来の戦闘機」ではなく、「静かな宇宙観測船」を意識する。

HUDには大量の数字、赤い警告、照準器、武器表示を置かない。

色は黒、濃紺、白を基本とし、情報表示には淡い水色や琥珀色を使う。

危険表示だけを赤色にする。

情報は必要なときだけ現れ、通常時は宇宙の景色を大きく見せる。

ゲーム全体を通して、急かされず、自分の速度で太陽系を旅できる雰囲気を重視する。

# 天体ビジュアル方針(実写ベーステクスチャ)

## 方針

実写ベースのテクスチャを使う方針を採用する。

### 1. 太陽・惑星・衛星は「実画像ベース」にする

* 太陽:実際の太陽観測画像をベースにしたテクスチャ
* 惑星:NASAなどの公開画像を元にした表面テクスチャ
* 衛星:月や主要衛星も、実際の観測画像や地形画像を元にしたテクスチャ

ただし、ゲーム内ではそのままベタ貼りではなく、見やすいように加工した素材にするのが大事。

### 2. 素材の入手元

一番安全なのは、NASA・USGS・JPL系の公開素材を使うこと。

候補は次の通り。

* NASAの惑星画像
* NASA Solar System Explorationの素材
* USGSの月面・惑星地形画像
* 木星や土星の衛星は探査機画像を元にした公開マップ
* 太陽はSDOなどの観測画像を元にしたビジュアル素材

大事なのは、「実際の画像を取り出して使う」＝そのまま雑にコピーするのではなく、公開利用条件を確認した上で、ゲーム用素材として整えること。

### 3. 実装上のおすすめ

天体画像は距離によって段階的に見え方を変えるのがよい。

* **遠距離表示**
  * 小さな円形スプライト
  * 色と明るさだけ分かる簡易表示
  * 必要なら環や大気だけ簡略表示
* **中距離表示**
  * 実画像ベースのテクスチャを貼った円形表示
  * 模様が見え始める
  * 月や衛星らしさが分かる
* **近距離表示**
  * より高解像度の画像へ切り替え
  * 大気のぼかし
  * 雲の層
  * 土星の環
  * 太陽の発光
  * 月面や火星表面の地形感

### 4. そのまま写真を使う時の注意

| 問題 | 対策 |
|---|---|
| 解像度が大きすぎる | 画像サイズを統一する |
| 色味がバラバラ | 明るさやコントラストを調整する |
| 背景が透過されていない | 黒背景を除去する |
| 天体によって向きや縮尺がバラつく | 円形に切り抜く |
| 2Dゲームでは細かすぎて逆に見にくい | 遠景用・近景用の複数サイズを作る |

### 5. まず実装すべき天体

最初から全部やると太陽系に負けるので、段階的にやるのがよい。

* **第1段階**: 太陽、水星、金星、地球、月、火星、木星、土星
* **第2段階**: 天王星、海王星、冥王星、火星の衛星、木星のガリレオ衛星、土星のタイタンなど主要衛星
* **第3段階**: さらに主要な衛星を拡充、天体ごとの説明表示、表面の違いを強調

最初は月・火星・木星・土星があるだけでも、かなり旅感が出る。

### 6. 天体ビジュアル仕様(まとめ)

* 太陽、惑星、衛星には、実際の観測画像や公開天体画像を元にしたテクスチャを使用する。
* 素材はNASA、USGS、JPLなどの公開利用可能な資料を優先する。
* ゲームでの視認性を高めるため、明度・色調・切り抜き・縮尺を調整した上で使用する。
* 表示は距離に応じて段階的に切り替える。
  * 遠距離:簡易スプライト表示
  * 中距離:標準テクスチャ表示
  * 近距離:高解像度テクスチャ表示
* 太陽は発光表現を加える。
* 惑星は大気、雲、環などの特徴を簡易エフェクトで補助表示する。
* 衛星は主要なものから順次対応する。

### 7. 実装でおすすめのフォルダ構成

```

assets/celestial/planets/earth/
assets/celestial/planets/mars/
assets/celestial/moons/moon/
assets/celestial/moons/europa/
```

各フォルダ内は例えば次のように構成する。

```
far.png
medium.png
near.png
meta.json
```

### 8. さらに相性のいい演出

実画像を使うなら、次も相性がよい。

* 天体に近づくと名前表示
* 簡単な解説テキスト
* 訪問記録へのスクリーンショット保存
* 月面や火星への着陸時に、表面の質感が見える
* 地球だけ少し特別に、青さと雲をきれいに見せる
